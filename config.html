Configuration

All Multivac components are configured via a common YAML configuration file.

/etc/multivac.yml example

#global config
redis: "127.0.0.1:6379"
actions:
  - name: echo
    cmd: /bin/echo
    chatbot_strem: True
    confirm_required: True

  - name: cool-command
    cmd: /usr/local/bin/mycoolscript.sh

#slackbot config
slack_token: 1234

#api config
api_listen_port: 8000

Configuration Values

  • redis(default: 127.0.0.1:6379): host and port of the redis instance to connect to
  • actions: A list of configured actions to be used in the creation of jobs. Actions accept the following parameters:
    • name

      • This serves as the command name to create the job via Slack message or API

    • cmd

      • Path of the command or script to execute

    • chatbot_stream(default: True)

      • If True, the output from the above command will be returned to the chat client. If False, the output of the above command will be stored but not sent back to the user as a chat response

    • confirm_required(default: False)

      • If True, a secondary confirmation will be required before the command is run. Default False

  • slack_token: Slack token used to connect to the Slack RTM API
  • api_listen_port(default: 8000): Port for which the API and web interface will listen
