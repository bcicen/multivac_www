API Documentation

This describes the components and design of Multivac. You can use the Quickstart to deploy Multivac.

Endpoints

/jobs

  • Get All Jobs
  • Get Job By ID
  • Create Job

/confirm

  • Confirm

/actions

  • Get Actions

/logs

  • Get Logs

/workers

  • Get Workers

/version

  • Get Version

GET /jobs

Returns a list of all jobs


Content-Type application/json
[
  {
    "args": "hello",
    "cmd": "/bin/echo",
    "confirm_required": "False",
    "created": "1440541057",
    "id": "1587bd4f70234c73852de387838b9434",
    "name": "echo",
    "status": "completed"
  }
]

GET /jobs/<job_id>

Returns a single job matching provided ID


Content-Type application/json
  {
    "args": "hello",
    "cmd": "/bin/echo",
    "confirm_required": "False",
    "created": "1440541057",
    "id": "1587bd4f70234c73852de387838b9434",
    "name": "echo",
    "status": "completed"
  }

POST /jobs

Create a new job and append it to the worker queue

Request


Content-Type application/json
  {
    "action": "echo",
    "action_args": "hello"
  }

Response


Content-Type application/json
{
    "id": "edba7a34a4b84d64b7ef69f47cf44748"
}

GET /confirm/<job_id>

Confirms a pending job

GET /actions

Returns a list of all configured actions


Content-Type application/json
[
  {
    "cmd": "/bin/echo",
    "confirm_required": "False",
    "name": "echo"
  }
]

GET /logs/<job_id>

Returns output log for provided job ID.
Optionally append ?json=true parameter to return logs in a JSON-formatted array


[Tue Aug 25 22:17:37 2015] hello
or 

Content-Type application/json
[
    "[Tue Aug 25 22:17:37 2015] hello"
]

GET /workers

Return a list of registered job workers


Content-Type application/json
[
    {
        "host": "localhost",
        "name": "Jacqueline"
    }
]

GET /version

Return Multivac version


Content-Type application/json
{
    "version": "v0.7"
}
